# Prompt 設計：產業導向的 ESG 內容生成

## 核心原則
**產業別是影響 LLM 生成內容的最關鍵因素，應該成為 prompt 的核心。**

## 角色設定
- **產業分析專家**：深度了解特定產業的營運模式、風險特徵、市場動態
- **ESG 專家**：掌握該產業的 ESG 關鍵議題、法規要求、最佳實務

## Prompt 結構設計

### 1. 執行長訊息 (CEO Message)

```
你是一位資深的 ESG 顧問和{產業別}產業分析專家。

請為一家{產業別}公司撰寫約 330 字的執行長訊息，用於 ESG 永續報告書。

**產業背景**：
- 產業：{產業別}
- 公司規模：{公司規模}
- 年營收：{營收}

**撰寫要求**：
1. 必須緊扣{產業別}的產業特性，例如：
   - {產業別}面臨的主要 ESG 挑戰
   - {產業別}的永續轉型機會
   - {產業別}利害關係人的核心關切

2. 平衡啟發性與責任感：歡迎讀者、概述永續願景、強調近期 ESG 成就、承認仍面臨的挑戰，並呼籲利害關係人共同參與轉型旅程。

3. 使用「我們」和「本公司」，保持溫暖且專業的語調，避免元評論。

**重要**：內容必須體現{產業別}產業的特色，而非通用模板。
```

### 2. 我們的公司 - 公司資訊

```
你是一位{產業別}產業分析專家和 ESG 顧問。

請為一家{產業別}公司撰寫約 345 字的公司介紹，用於 ESG 報告。

**公司資訊**：
- 產業：{產業別}
- 公司規模：{公司規模}
- 年營收：{營收}

**撰寫要求**：
1. 第一句使用 {COMPANY_NAME} 作為公司名稱佔位符。

2. 必須突出{產業別}產業的特色：
   - {產業別}的商業模式特徵
   - {產業別}的價值鏈位置
   - {產業別}的競爭優勢來源

3. 總結背景、商業模式、地理足跡、策略夥伴關係和組織架構。

4. 強調使命、價值觀，以及合作如何支撐長期競爭力。

**重要**：內容必須反映{產業別}產業的實際營運情境。
```

### 3. 我們的公司 - 財務資訊

```
你是一位{產業別}產業財務分析專家和 ESG 顧問。

請為一家{產業別}公司撰寫約 345 字的財務穩定性說明，用於 ESG 報告。

**公司財務背景**：
- 產業：{產業別}
- 公司規模：{公司規模}
- 年營收：{營收}

**撰寫要求**：
1. 必須結合{產業別}產業的財務特徵：
   - {產業別}常見的財務風險
   - {產業別}的資本密集度
   - {產業別}的現金流特性

2. 詳細說明財務穩定性、營收成長、資本投資和法規遵循。

3. 討論風險管理、流動性紀律、再投資優先順序和保證機制。

4. 保持自信且透明的語調，避免元評論。

**重要**：財務分析必須符合{產業別}產業的實際情況。
```

### 4. 利害關係人分析

```
你是一位{產業別}產業分析專家和 ESG 顧問。

請為一家{產業別}公司撰寫約 375 字的利害關係人分析，用於 ESG 報告。

**產業背景**：
- 產業：{產業別}
- 公司規模：{公司規模}

**撰寫要求**：
1. 必須識別{產業別}產業特有的利害關係人群體：
   - {產業別}的核心利害關係人（例如：製造業的供應商、服務業的客戶、金融業的監管機構）
   - {產業別}特有的利害關係人關切議題

2. 分析每個群體的重要性、期望、若被忽視的潛在風險，以及如何設定優先順序。

3. 涵蓋顯著性（權力、合法性、緊迫性）、重大議題、溝通節奏，以及監控參與成效的關鍵績效指標。

4. 保持高階主管語調，使用簡潔的中文，避免項目符號。

**重要**：利害關係人分析必須針對{產業別}產業量身定制。
```

### 5. 重大議題

```
你是一位{產業別}產業 ESG 專家和風險管理顧問。

請為一家{產業別}公司撰寫約 375 字的重大議題總結，用於 ESG 報告。

**產業背景**：
- 產業：{產業別}
- 公司規模：{公司規模}

**TCFD 分析結果**（如有）：
- 政策法規風險：{tcfd_policy_context}
- 市場趨勢風險：{tcfd_market_context}

**撰寫要求**：
1. 必須聚焦{產業別}產業的重大 ESG 議題：
   - {產業別}最常見的 ESG 風險
   - {產業別}的監管重點
   - {產業別}的市場轉型壓力

2. 說明優先順序設定方法、與策略和風險的連結、利害關係人意見，以及預期成果。

3. 如果提供了 TCFD 分析結果，必須整合這些資訊，但要以{產業別}產業的角度來解讀。

4. 保持分析性且易於理解的語調，不使用元評論或項目列表。

**重要**：重大議題必須反映{產業別}產業的實際 ESG 挑戰。
```

### 6. ESG 核心支柱

```
你是一位{產業別}產業 ESG 專家和永續發展顧問。

請為一家{產業別}公司撰寫約 375 字的 ESG 核心支柱說明，用於 ESG 報告。

**產業背景**：
- 產業：{產業別}
- 公司規模：{公司規模}
- 年碳排放：{emission_context}

**撰寫要求**：
1. 必須說明{產業別}產業在 ESG 三大支柱（地球、產品、人員）的重點：
   - {產業別}在地球（Planet）方面的關鍵議題
   - {產業別}在產品（Products）方面的永續機會
   - {產業別}在人員（People）方面的管理重點

2. 討論重點領域、跨價值鏈的整合、衡量實務和問責機制。

3. 提及表格顯示每個支柱的摘要和倡議。

4. 使用簡潔的中文，避免項目符號。

**重要**：ESG 支柱的說明必須針對{產業別}產業的特性。
```

### 7. 風險管理

```
你是一位{產業別}產業風險管理專家和 ESG 顧問。

請為一家{產業別}公司撰寫約 375 字的風險管理方法概述，用於 ESG 報告。

**產業背景**：
- 產業：{產業別}
- 公司規模：{公司規模}

**TCFD 分析結果**（如有）：
- 政策法規風險：{tcfd_policy_context}

**撰寫要求**：
1. 必須說明{產業別}產業特有的風險管理重點：
   - {產業別}常見的營運風險
   - {產業別}的監管風險
   - {產業別}的市場風險

2. 涵蓋治理結構、評估節奏、緩解規劃、控制監控、升級和保證。

3. 如果提供了 TCFD 分析結果，必須整合這些資訊，但要以{產業別}產業的風險管理框架來解讀。

**重要**：風險管理方法必須符合{產業別}產業的實際風險特徵。
```

## 關鍵改進點

### 1. 產業別成為核心
- 每個 prompt 都以「你是一位{產業別}產業分析專家和 ESG 顧問」開頭
- 明確要求「必須緊扣{產業別}的產業特性」
- 強調「內容必須體現{產業別}產業的特色，而非通用模板」

### 2. 減少資料依賴
- 只使用核心資訊：產業別、公司規模、營收
- TCFD 資料作為補充，而非主要依賴
- 移除過長的背景資料

### 3. 專家角色設定
- 產業分析專家：提供產業深度洞察
- ESG 專家：提供 ESG 專業知識
- 兩者結合：確保內容既符合產業特性，又符合 ESG 標準

### 4. 明確指示
- 「必須」的強制性要求
- 「重要」的強調事項
- 具體的產業特性要求

## 實施建議

1. **產業別映射**：建立產業別到關鍵特性的映射表
2. **動態 prompt**：根據產業別動態生成 prompt
3. **範例庫**：為不同產業建立範例，供 LLM 參考
4. **驗證機制**：檢查生成內容是否真正體現產業特色
